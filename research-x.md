# research-x

Knowledge layer for research. Papers live in Zotero; deep notes live here.

## Three-Pass Reading System

Papers go through up to three passes, each with a different depth and output destination.

| Pass | What happens | Output | Where |
|---|---|---|---|
| 1 — Triage | Me + Claude, quick read and discussion | Short memo — key claim, my take | Zotero note on the item |
| 2 — Deep read | Me + Claude, detailed walkthrough | Full note — methods, evidence, questions | `odyssey` repo (github.com/ldsun/odyssey) |
| 3 — Hands-on | Reproduction, experimentation | Out of scope for this system | — |

### Pass 1 — What Claude does

1. Read the paper via MCP (abstract + key sections)
2. Discuss with me; synthesize my understanding into a compact memo
3. Write memo as a note on the Zotero item
4. Apply content tags to the item (see Tags below)
5. Add item to `passes/pass-1` collection (create if missing)
6. If worth a deep read → also add to `passes/pass-2` as a queue signal

### Pass 2 — What Claude does

1. Read the full paper deeply via MCP
2. Work through methods, mechanisms, and evidence with me
3. Write detailed note to `odyssey` repo (`github.com/ldsun/odyssey`)
4. Add item to `passes/pass-2` collection in Zotero

## Zotero Organization

### Collections

```
passes/
  pass-1      ← memo written
  pass-2      ← deep read done
  pass-3      ← hands-on done (future)
```

Collections track pass status only. Claude creates subcollections on first use if they don't exist.

### Tags

Pure content taxonomy — what the paper is about, in standard industry/academic vocabulary. Generated by Claude from the paper itself. No personal or workflow signal.

Examples: `3d-reconstruction`, `self-supervised-learning`, `object-detection`, `multi-object-tracking`, `point-cloud`, `sensor-fusion`, `vision-language-model`, `autonomous-vehicles`, `robotics`, `sports`, `dataset`, `benchmark`, `annotation-quality`

**To filter:** click `passes/pass-2` collection → filter by tag in the left panel.

Tags go on the **item** (not the note) so they appear in Zotero's tag cloud and sidebar filter.

## How the Bridge Works (MCP)

Claude reads and writes Zotero via a local MCP server (`zotero-mcp`) running on Mac Mini:

```
Claude Code (VS Code, Mac Mini)
    ↕ MCP (stdio)
zotero-mcp process
    ↕ HTTP
Zotero local API (localhost:23119)   +   Zotero web API (api.zotero.org)
    ↕
Zotero app (must be running on Mac Mini)
```

| Operation | Requires |
|---|---|
| Search, read metadata, read full PDF | Local mode (no key needed) |
| Write notes, apply tags, manage collections | API key (hybrid mode) |

## research-x Structure

```
research-x/
  README.md
  inbox.md              ← raw reading notes → processed by /process
  notes/
    papers/             ← one .md per paper (pass-2 deep notes)
    topics/             ← synthesis across papers
```

### Per-paper note (`notes/papers/`) — pass 2 output

- Problem the paper solves
- Key contributions
- Methods / approach in detail
- Key results / numbers
- Questions / follow-ups
- Connections to other papers

### Topic note (`notes/topics/`)

- Synthesis across papers on a theme
- What do I believe?
- Open questions
